# 这个模块是干什么的？🤔
总所周知，牢米在miui13及之后的澎湃系列里面，删除了许多功能【包括哈曼卡顿（这个可以理解，因为事实上合约就那几年，而且貌似只有XIAOMI 10s是有约束的），声音视效（我反正觉得很难绷，只是因为耗电就把它砍了？这个理由多少有点不充分，而且太过抽象了，并非是没有给你关闭的按钮，这东西按理来说也不应该耗电到哪去）】

但是相对于删除，事实上这更像是隐藏【因为你可以在anywhere和创建快捷方式里面看到，牢米甚至没有将这两个activity给“划掉”，而是将其从“高亮”状态（即用户可使用常规方式，在应用内开启）切换为“白底”状态（用户无法在应用内看到，但是activity仍然可以通过adb或是root的形式打开）。但是有一些用户的设备（一部分性价比“弃舰”，比如狂暴引擎限时体验卡，相当出名的note12turbo和早期的一些k系列旗舰（是不是至尊版没有差别，该被放弃还是会被放弃的）），是没有这两个activity的（对的，兄弟，你没有听错，牢米直接把这两个activity从一些设备上移除了），这一部分设备就比较尴尬了，因为如果不对系统进行降级的话，即便root也永远用不了这些功能】。

于是这个模块就应运而生☝🏻🤓：通过加入prop值，来强制开启哈曼卡顿和声音视效的开关。（对于上面被直接移除activity的设备也应该有效）

# 模块具体会对我的手机做什么？🤔
>[!IMPORTANT]
>模块会对你的手机做下面三件事☝🏻🤓：

**1.添加系统的prop值："ro.vendor.audio.sfx.harmankardon=true",这一条是用来开启哈曼卡顿的。**

**2.也是添加系统prop值："ro.vendor.audio.sfx.audiovisual=true",这一条是用来开启声音视效的。**

**3.添加文件：向目录"/system/etc/audio"内添加四个声音视效视频，实现声音视效的完整开启。**

# 这个模块会导致一些其他的问题嘛？🤔
有可能，在刷入前请检查你的系统。

如果你的系统不够完整（包括线下机和走量机，低端机，比如红米的note数字系列，红米数字系列等），或者你的系统采用的是其他社区开发者的精简版系统，那么模块没有生效的情况将有可能是家常便饭。即便模块生效，也可能会导致检测问题，包括native test和hunter等检测软件可能提示prop被修改。因此，在刷入前请检查你的系统，确保功能完整的同时，不会引发其他兼容性问题。

如果想要检查情况，可以这么做：

1.下载mt管理器，然后打开终端模拟器/下载termux，打开

2.然后输入下面的内容：（一个一个来，把超级用户给到）

su

getprop ro.vendor.audio.sfx.harmankardon

getprop ro.vendor.audio.sfx.audiovisual

如果你的结果是true，请检查你的音质音效是否出现两个功能。没有出现的话，那非常抱歉，你的系统看起来并不支持❌（这怎么说呢？有好几种情况，有的是系统不认这个prop值；有的是认了，但是被某些东西给干扰了也有可能），如果可以的话，还请换个系统。

如果你的结果是false/空值/不存在，那请你继续往下看。
# 老师，我刷入了模块，但是它看起来并没有生效，这是什么情况？🤔
情况非常复杂，有的机型是有功能没效果，有的是有效果没开关。在刷入后没有效果的情况下，则请先用上面检查系统的办法进行测试，出现false或是空的话再往下看。

>[!TIP]
>很好，现在结果是false/空值/不存在，模块没有生效，这一般是因为模块的工作异常导致的（当然也有可能由于挂载等等一些其他问题）针对于没有生效的情况，可以这么做：

**1.寻找一个有system.prop的模块。**

**2.把位于system.prop的代码加入模块中的system.prop。**

**3.把位于service.sh的代码加入模块的service.sh（最好进行修改，否则有可能导致另一模块异常工作）。**

**4.将system文件夹附加到模块中。**

**5.重启设备。**

这么做将会使模块寄生在其他模块内，或者叫做对其他模块的功能补充，同样也可正常工作（在我的设备上正常运行）。

当然，如果你不附加service.sh中的代码，理论上同样没有问题（但是如果出现模块不生效或是异常运行，可能无法有效解决，因为缺少日志描述，无法确认问题出现的情况）
